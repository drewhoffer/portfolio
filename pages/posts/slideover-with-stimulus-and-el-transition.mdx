# How to Use StimulusJS to Create a Slideover in Your Rails Apps
If you're wanting to use Rails but confused how to add interactivity to an app, overwhelmed by TailwindUI's templates for html then look no further.
Today we're going to add a sleek, animated slideover to your Rails app. Let's dive in!

---

## 1. Set Up the Rails App

First, create a new Rails app with StimulusJS and ESBuild (easier for newcomers than importmaps):

```bash
rails new stimulus_slideover --css tailwind --javascript esbuild
```

---

## 2. Add a Home View

Generate a simple `home` view and controller:

```bash
rails g controller home index
```

Update the root route to point to the new view:

```ruby
# config/routes.rb
Rails.application.routes.draw do
  root "home#index"
end
```

This sets up a playground for our slideover. Run the app and let's get started!

---

## 3. Add the Slideover UI

For this example, we'll use the free TailwindUI slideover template from [here](https://tailwindui.com/components/application-ui/application-shells/sidebar#component-73d9b2c2e2af9184f305f06649710086). Be sure to copy the **HTML version** (I'm using Tailwind v3.4).

Update the application layout to remove any default padding from the installation `application.html.erb`:

```html
<!DOCTYPE html>
<html class="h-full bg-white">
  <head>
    <title>Stimulus Slideover</title>
```

## 4. Create the Slideover Stimulus Controller

Generate a Stimulus controller for the slideover:

```bash
rails g stimulus slideover
```

---

## 5. Hook Up the HTML and Controller

In your `home/index.html.erb`, you'll notice the TailwindUI template includes comments about transitions. Don’t worry about those for now; we’ll handle animations later with a library called `el-transition`.

Start by ensuring the slideover doesn’t load in an open state. Add the Stimulus controller to your markup:

```html
<div data-controller="slideover">
  <!-- Off-canvas menu for mobile, show/hide based on off-canvas menu state. -->
  <div class="relative z-50 lg:hidden" role="dialog" aria-modal="true">
    <!-- Trimmed for brevity -->
  </div>
</div>
```

Here’s the breakdown:

- **`data-controller="slideover"`**: Attaches the Stimulus controller to the slideover.
- Include the navbar in this scope so the button can toggle the slideover.

---

## 6. Define Stimulus Targets

Identify the key components of the slideover:

- The backdrop
- The content of the slideover
- The `contentAnchor`, wrapping the entire slideover

Add these as targets in the Stimulus controller:

```javascript
// app/javascript/controllers/slideover_controller.js
import { Controller } from "@hotwired/stimulus";

// Connects to data-controller="slideover"
export default class extends Controller {
  static targets = ["backdrop", "content", "contentAnchor"];

  connect() {
    console.log("Hello slideover!");
  }
}
```

If everything is set up correctly, refreshing the page should log `Hello slideover!` in the console.

---

## 7. Add Targets to the Markup

Now that the targets are defined, update your markup to include them:


In short:
```html
<div data-controller="slideover">
  <div data-slideover-target="backdrop" class="fixed inset-0 bg-gray-500 bg-opacity-75"></div>
    <div data-slideover-target="contentAnchor" class="fixed inset-0 flex">
		<div data-slideover-target="content" class="relative mr-16 flex w-full max-w-xs flex-1">
			<!-- More slideover content here -->
		</div>
  </div>
  <!-- Navbar here INSIDE the slideover controller -->
	<nav>
		<!-- ... -->
		 <!-- Button that opens the slideover -->
	</nav>
</div>
```

## 8. Add Visibility Control

Now that we have the parts targeted, let’s start by hiding them on `connect`.

Update the Stimulus controller:

```js
import { Controller } from "@hotwired/stimulus";

// Connects to data-controller="slideover"
export default class extends Controller {
  static targets = ["backdrop", "content", "contentAnchor"];

    this.backdropTarget.classList.add("hidden");
    this.contentTarget.classList.add("hidden");
    this.contentAnchorTarget.classList.add("hidden");
```

This hides the slideover components on page load.

## 10. Add Animations with `el-transition`

Install `el-transition`:

```bash
npm install el-transition
```

Update the controller to use `el-transition` for animations:

```js
import { Controller } from "@hotwired/stimulus";
import { enter, leave } from "el-transition";

// Connects to data-controller="slideover"
export default class extends Controller {
  // Targets are the elements we want to do something to, you'll notice we dont have
  // the opener button, thats because we dont need to toggle its visibility
  // like we do with the close button
  static targets = ["backdrop", "content", "contentAnchor"];
  connect() {
    console.log("Hello slideover!");
    // toggle hidden on backdrop and content
    this.backdropTarget.classList.add("hidden");
    this.contentTarget.classList.add("hidden");
    this.contentAnchorTarget.classList.add("hidden");
  }

  openSlideover() {
    // remove hidden from backdrop and content
    enter(this.backdropTarget);
    enter(this.contentTarget);
    enter(this.contentAnchorTarget);
  }

  async closeSlideover() {
    await Promise.all([leave(this.backdropTarget), leave(this.contentTarget)]);
    this.contentTarget.classList.add("hidden");
    this.contentAnchorTarget.classList.add("hidden");
    this.backdropTarget.classList.add("hidden");
  }
}
```
